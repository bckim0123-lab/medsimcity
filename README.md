# MediSim — 범용 지역 맞춤형 의료 정책 시뮬레이터

> HIRA 데이터 기반 2D/3D 입지 적합성 및 실효성 분석 엔진

## 개요

MediSim은 지자체 보건행정 담당자가 신규 공공 의료시설(보건소, 달빛의원, 심야약국 등)의 최적 입지를 과학적으로 결정하고, 정책 효과를 사전에 검증할 수 있는 **B2G SaaS 플랫폼**입니다.

### 핵심 가치
| 목표 | 수단 |
|------|------|
| 미충족 의료 수요 발굴 | 격자 기반 이동시간 공백 분석 |
| 골든타임 확보 | 도로망 기반 15분 커버리지 계산 |
| 예산 효율화 | 중복도(Redundancy) 자동 경고 |
| 데이터 주권 | 100% 오픈소스, Zero-Cost 파이프라인 |

## 빠른 시작

### 요구사항
- Python 3.11+
- Node.js 18+

### 1. 백엔드 실행

```powershell
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload --port 8000
```

서버가 시작되면 자동으로 SQLite DB를 생성하고 서울 의료시설 Mock 데이터를 시드합니다.

API 문서: http://localhost:8000/docs

### 2. 프론트엔드 실행

```powershell
cd frontend
npm install   # 이미 설치된 경우 생략
npm run dev
```

브라우저에서 http://localhost:3000 접속

## 주요 기능

### F1. 지역 프로파일링
- 사이드바 검색창에 지역명 입력 (예: "강남구", "노원구")
- Nominatim(OSM 무료 지오코더)로 자동 이동
- 인구 밀도 헥사곤 레이어 + 의료시설 마커 렌더링

### F2. 의료 공백 분석 (Gap Analysis)
- 상단 탭 → **의료 공백 분석** 선택
- 200~500m 격자 단위로 최근접 시설까지의 이동시간 계산
- 색상 범례: 초록(양호) → 노랑 → 주황 → 빨강(심각)
- 질환 유형 선택 시 역학적 가중치 자동 적용:
  - 응급: 골든타임 10분 기준
  - 만성질환: 고령자 인구 가중
  - 소아청소년: 아동 인구 가중

### F3. 시나리오 에디터
1. 상단 탭 → **시나리오 편집** 선택
2. 붉은 격자(의료 취약지)를 클릭하여 후보지 배치 (★ 마커)
3. KPI 패널 → **분석 실행** 클릭
4. 결과 지표 확인:
   - 정책 실효성 점수 (0~100)
   - 커버리지 인구 증가수
   - 평균 이동시간 단축율
   - 예산 낭비 위험 경고 (중복도)

## 스코어링 알고리즘

```
정책 실효성 점수 = (접근성 점수 × 0.5 + 커버리지 점수 × 0.5 - 중복 패널티) × 역학적 가중치
```

| 변수 | 계산 방식 |
|------|-----------|
| 접근성 점수 | 평균 이동시간 감소율 × 2 (capped at 100) |
| 커버리지 점수 | 신규 15분 이내 인구 / 전체 인구 × 500 |
| 중복 패널티 | 기존 시설 800m 이내 배치 시 -25점 |
| 역학적 가중치 | 응급 1.5× / 소아 1.3× / 만성 1.0× |

## 기술 스택

| 영역 | 기술 |
|------|------|
| 지도 렌더링 | MapLibre GL JS (무료, API 키 불필요) |
| 데이터 시각화 | Deck.gl v9 (HexagonLayer, GeoJsonLayer) |
| 프론트엔드 | Next.js 14 + TypeScript + Tailwind CSS |
| 상태 관리 | Zustand |
| 백엔드 | Python FastAPI + SQLAlchemy |
| 데이터베이스 | SQLite (운영환경: PostgreSQL 전환 권장) |
| 공간 분석 | scipy.spatial.KDTree |
| 지오코딩 | OSM Nominatim (무료) |
| 지도 타일 | CARTO Dark Matter (무료) |

## 실제 HIRA 데이터 연동

MVP에서는 Mock 데이터를 사용합니다. 실제 배포 시 다음 단계로 전환:

1. [공공데이터포털](https://www.data.go.kr) → "전국 병·의원 및 약국 현황 API" 신청
2. `backend/app/seed.py`의 `seed_data()` 함수를 HIRA API 호출 코드로 교체
3. 주 1회 Celery 태스크로 DB 자동 동기화
